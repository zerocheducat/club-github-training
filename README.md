# 🧑‍💻 新人“破冰”任务

欢迎加入团队！(oﾟ▽ﾟ)o

在你深入研究代码库之前，我们为你准备了一个简单（但直击灵魂）的小任务。这既是为了让你熟悉我们使用 Markdown 的基本流程，也是为了让我们（用一种不那么严肃的方式）了解你。

## 你的任务：一篇“小作文”

请创建一个新的 Markdown 文件，完成一篇短文。

**题目如下（请以此为标题）：**

> 《我为什么想不开要学计算机》

是的，你没看错。

## 写作要求

* **灵魂拷问：** 请真实（或半真实，我们不戳穿）地回顾一下你的心路历程。是因为热爱？是“听说”的高薪？是高考调剂？还是单纯因为《黑客帝国》看起来很酷？
* **格式要求：**
    * 必须使用 **Markdown 格式** 书写。
    * 至少使用到以下 5 种 Markdown 语法：
        1.  **标题** (例如 `#` 或 `##`)
        2.  **加粗** (`**文字**`) 或 *斜体* (`*文字*`)
        3.  **无序列表** (`*`) 或 **有序列表** (`1.`)
        4.  **引用** (`>`)
        5.  **代码块** (用 \`\`\` 包裹，哪怕只是写一句 `Hello World`)
* **字数：** 不限，但至少 300 字，让我们感受到你的“真诚”。

## 提交指南

1.  在本项目的 `newcomers` 目录下（如果没有就创建一个）。
2.  以你的 **用户名** 命名该 Markdown 文件，例如：`zhangsan.md`。
3.  完成写作后，提交一个 Pull Request (PR)。
4.  在 PR 的描述中 @ 你的导师（Mentor）。
